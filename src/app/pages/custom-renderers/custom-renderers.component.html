Custom Cell Renderers

Create custom components for specialized cell rendering

Custom Renderers in Action
Profile column shows custom avatar initials. Status column uses colored badges. Actions column has clickable buttons.

Custom Renderer Concepts

Creating a Custom Renderer

A cell renderer is an Angular component with parameters property for cell data. Implement the method agInit(params) to receive cell data.

Renderer Component (profile-picture-renderer.component.ts): 
import {{ Component }} from '@angular/core';

@Component({{
  selector: 'app-profile-picture-renderer',
  template: `<div class="profile-avatar">
    {{ params.value?.substring(0, 2).toUpperCase() }}
  </div>`,
  styles: [`...`]
}})
export class ProfilePictureRendererComponent {{
  params: any;

  agInit(params: any) {{
    this.params = params;
  }}

  refresh() {{
    return true;
  }}
}}

Registering in Column Definition

Add the renderer component to cellRenderer property. The grid will instantiate and pass cell data automatically.

Column Definition: 
{{
  headerName: 'Profile',
  field: 'profile',
  cellRenderer: ProfilePictureRendererComponent
}}

Accessing Cell Data

The params object contains all cell information.

Available Properties: 
params.value           // Cell value
params.data            // Row data object
params.rowIndex        // Row number
params.colDef          // Column definition
params.api             // Grid API
params.columnApi       // Column API

Status Renderer Example

Render conditional badges based on cell value.

Status Renderer Template: 
<span [ngClass]="{
  'badge-success': status === 'Active',
  'badge-danger': status === 'Inactive',
  'badge-warning': status === 'Pending'
}">
  {{ status }}
</span>

Action Buttons Renderer

Create interactive buttons within cells. Emit events to parent component.

Actions Renderer Template: 
<button class="btn btn-sm btn-primary" (click)="onEdit()">
  Edit
</button>
<button class="btn btn-sm btn-danger" (click)="onDelete()">
  Delete
</button>

Refresh Method

Implement refresh() method to update renderer when cell data changes. Return true to trigger re-render.

TypeScript: 
refresh(): boolean {{
  // Update component state based on new params
  return true; // Renderer will update
}}

Best Practices

✓ Keep renderers simple: Complex components impact performance.
✓ Declare in module: Add to declarations and entryComponents for AOT.
✓ Implement refresh(): Handle data updates efficiently.
✓ Use for presentation only: Don't store component state.
✓ Test in isolation: Renderers should work independently.
