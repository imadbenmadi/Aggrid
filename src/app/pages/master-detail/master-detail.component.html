Master/Detail Grid

Display nested grids with hierarchical data

Master/Detail Implementation
Click the expand arrow (►) on any row to see related detail data in a nested grid. Each employee shows their skills in a detail grid.

Master/Detail Concepts

Enabling Master/Detail

Set [masterDetail]="true" on the grid. Add an expand column automatically. Use detailRowHeight to set height of detail rows.

HTML Template: 
<ag-grid-angular
  [masterDetail]="true"
  [detailRowHeight]="300"
        [detailCellRendererParams]="detailCellRendererParams"
  ...
>
</ag-grid-angular>

Detail Cell Renderer

Create a template reference for the detail view. This template is rendered for each expanded row.

HTML Template: 
<ng-template #detailCellRenderer let-data>
  <div>
    <h5>Details for {{ data.name }}</h5>
    <p *ngFor="let item of data.details">
      {{ item }}
    </p>
  </div>
</ng-template>

Loading Detail Data

Use getDetailRowData to provide data to the detail view. This function is called when a row is expanded.

TypeScript: 
getDetailRowData = (params: any) => {{
  // params.data contains the master row data
  // params.successCallback passes data to detail view
  params.successCallback(params.data.details);
}};

Alternative: Nested Grid

For more complex detail views, you can embed another AG Grid inside the detail renderer.

HTML Template: 
<ng-template #detailCellRenderer let-data>
  <ag-grid-angular
    [rowData]="data.details"
    [columnDefs]="detailColumnDefs"
    ...
  >
  </ag-grid-angular>
</ng-template>

Best Practices

✓ Load detail data on demand: Only fetch detail data when a row is expanded.
✓ Limit detail rows: Keep detail content reasonably sized for performance.
✓ Use nested grids carefully: Multiple nested grids can impact performance.
✓ Provide clear visual hierarchy: Use distinct styling for detail sections.
✓ Handle async data: Use promises or observables to load detail data.
