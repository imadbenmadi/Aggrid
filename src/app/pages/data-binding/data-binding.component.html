Data Binding Concepts

One-Way Binding

Use property binding [rowData]="rowData" to pass data to the grid. When you modify the array, Angular's change detection automatically updates the grid.

HTML: 
<ag-grid-angular [rowData]="rowData" />

Adding New Rows

To add a new row, create a new object and add it to the rowData array. Use immutable patterns to ensure Angular detects the change.

TypeScript: 
addNewRow() {{
  const newRow = {{
    id: newId,
    name: this.newName,
    age: this.newAge,
    // ... other properties
  }};

  // Immutable update - creates new array reference
  this.rowData = [...this.rowData, newRow];
}}

Updating Existing Rows

Map through the array to update specific rows. Always create a new array reference so Angular's change detection recognizes the modification.

TypeScript: 
updateRow(id: number, updates: any) {{
  this.rowData = this.rowData.map(row =>
    row.id === id ? {{ ...row, ...updates }} : row
  );
}}

Deleting Rows

Filter out rows to delete them. This creates a new array reference that triggers the grid to re-render.

TypeScript: 
deleteRow(id: number) {{
  this.rowData = this.rowData.filter(row => row.id !== id);
}}

Replacing All Data

Sometimes you need to reload all data from a server. Simply assign the new array to trigger a full grid refresh.

TypeScript: 
async reloadData() {{
  const newData = await this.apiService.fetchData();
  this.rowData = newData;
}}

Immutability Pattern

Always create new array references when modifying data. This is crucial for Angular's change detection and performance optimization strategies.

Good (Immutable): 
this.rowData = [...this.rowData, newRow];
this.rowData = this.rowData.map(...);
this.rowData = this.rowData.filter(...);
Bad (Mutable): 
this.rowData.push(newRow);  // Don't do this!
this.rowData[0].name = 'New';  // Don't do this!

Best Practices

✓ Use immutable patterns: Always create new array references when modifying data.
✓ Batch updates: Update multiple rows at once instead of one at a time.
✓ Optimize large updates: Use grid API methods for better performance with large datasets.
✓ Handle async operations: Use async/await or RxJS for API calls.
✓ Validate data: Always validate before adding or updating rows.
