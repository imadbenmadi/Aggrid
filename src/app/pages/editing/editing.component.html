Cell Editing Concepts

Enabling Cell Editing

Set editable: true on a column to allow editing. The default editor is a text input. Double-click or press Enter to edit.

Example: 
{{
headerName: 'Name',
field: 'name',
editable: true
}}

Cell Editors

Use different cell editors for different data types:

  • agTextCellEditor: Text input (default)
  • agSelectCellEditor: Dropdown selection
  • agRichSelectCellEditor: Rich select dropdown
  • agNumberCellEditor: Number input
  • agDateCellEditor: Date picker
Select Editor Example: 
{{
headerName: 'Department',
field: 'department',
editable: true,
cellEditor: 'agSelectCellEditor',
cellEditorParams: {{
  values: ['Engineering', 'Sales', 'Marketing', 'HR']
}}
}}

Detecting Changes

Listen to cellValueChanged event to react when a cell is edited.

TypeScript: 
onCellValueChanged(event: any) {{
  console.log('Old Value:', event.oldValue);
  console.log('New Value:', event.newValue);
  console.log('Row Data:', event.data);
}}

Programmatic Editing

Start and stop editing programmatically using the Grid API.

TypeScript: 
startEditing() {{
  this.gridApi?.startEditingCell({{
    rowIndex: 0,
    colKey: 'name'
  }});
}}

stopEditing() {{
  this.gridApi?.stopEditing();
}}

Edit Types

Choose between fullRow and singleCell editing modes:

  • singleCell: Edit one cell at a time
  • fullRow: Edit entire row at once (default)

Best Practices

✓ Validate input: Use valueParser to validate cell values before saving.
✓ Save to backend: Persist changes to your API/database.
✓ Show edit mode: Use cellStyle to highlight editable cells.
✓ Limit editable columns: Only make necessary columns editable.
✓ Use appropriate editors: Match editor to data type.
