Enabling Sorting

Column Sorting

Add sortable: true to column definitions to enable sorting. Users can click the column header to sort, and click again to reverse order.

Example: 
{{
headerName: 'Salary',
field: 'salary',
sortable: true
}}

Programmatic Sorting

Use the Column API to set sort order programmatically. Pass 'asc', 'desc', or null to the setSort() method.

TypeScript: 
sortBySalaryDesc() {{
  this.gridColumnApi
    ?.getColumn('salary')
    ?.setSort('desc');
}}

Enabling Filtering

Filter Types

AG Grid supports several filter types. Choose based on your data type:

  • agTextColumnFilter: Text/string filtering with contains, equals, etc.
  • agNumberColumnFilter: Numeric filtering with comparisons
  • agDateColumnFilter: Date filtering with date ranges
  • agSetColumnFilter: Choose from distinct values (like a multi-select)
Example: 
{{
headerName: 'Department',
field: 'department',
filter: 'agSetColumnFilter'  // Shows unique values
}}

Programmatic Filtering

Use the Grid API to apply filters programmatically. Set a filter model to apply filters without user interaction.

TypeScript: 
filterEngineering() {{
  const filterModel = {{
    department: {{
      type: 'set',
      values: ['Engineering']
    }}
  }};
  this.gridApi?.setFilterModel(filterModel);
}}

Number Filter Example

For numeric filters, specify the operator and value. Available operators: greaterThan, lessThan, equals, etc.

TypeScript: 
filterHighSalary() {{
  const filterModel = {{
    salary: {{
      type: 'numberFilter',
      operator: 'greaterThan',
      value: 80000
    }}
  }};
  this.gridApi?.setFilterModel(filterModel);
}}

Clearing Filters

Pass null or an empty object to clear all filters.

TypeScript: 
clearFilters() {{
  this.gridApi?.setFilterModel(null);
}}

Best Practices

✓ Enable sorting by default: Most users expect to sort columns.
✓ Match filter type to data: Use appropriate filters for strings, numbers, and dates.
✓ Disable sorting for calculated columns: Set sortable: false for columns computed on the fly.
✓ Consider performance: With large datasets, server-side sorting/filtering is better.
✓ Provide presets: Offer common filter/sort combinations for better UX.
