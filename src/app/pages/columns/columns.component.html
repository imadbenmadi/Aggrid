Columns & Cell Rendering

Master column definitions and customize cell appearance

Advanced Column Configuration
This grid demonstrates various column properties including filtering, formatting, styling, and pinning.

Column Definition Properties

Basic Properties

  • field: The property name from your data object
  • headerName: The text shown in the column header
  • width: Column width in pixels
  • hide: Boolean to show/hide the column
Example: 
{{
headerName: 'Name',
field: 'name',
width: 200,
hide: false
}}

Pinning Columns

Use the pinned property to lock columns in place during horizontal scrolling. Useful for ID columns or important reference fields.

Example: 
{{
headerName: 'ID',
field: 'id',
pinned: 'left',  // or 'right'
width: 80
}}

Cell Styling

Use cellStyle to apply inline styles to cells. Can be a static object or a function that receives the cell context and returns dynamic styles.

Example: 
{{
headerName: 'Status',
field: 'status',
cellStyle: params => ({{
  'background-color': params.value === 'Active' ? '#d4edda' : '#f8d7da',
  'text-align': 'center'
}})
}}

Value Formatter

Format cell values for display without changing the underlying data. Common for currency, dates, and percentages.

Example: 
{{
headerName: 'Salary',
field: 'salary',
valueFormatter: params => '$' + params.value?.toLocaleString()
}}

Column Filtering

Enable filtering with different filter types: agTextColumnFilter, agNumberColumnFilter, agDateColumnFilter, or agSetColumnFilter.

Example: 
{{
headerName: 'Department',
field: 'department',
filter: 'agSetColumnFilter'  // Shows unique values
}}

Column Types

Use column types to apply preset configurations. Built-in types: numericColumn, rightAligned, dateColumn. They set common properties automatically.

Example: 
{{
headerName: 'Age',
field: 'age',
type: 'numericColumn'  // Right-aligned, numeric filter
}}

Best Practices

✓ Use valueFormatter for display: Keep original data unchanged while showing formatted values.
✓ Pin important columns: Keep ID or reference columns visible during scrolling.
✓ Match data types: Use appropriate filter types (numeric, text, date, set).
✓ Use column types: Leverage built-in types to avoid repetitive configuration.
✓ Dynamic styling: Use functions for cellStyle to apply conditional formatting.
