Data Grouping

Organize and summarize data by grouping rows

Grouped Data
Data is grouped by Department. Expand groups to see individual records. Note: Salary is summed for each group.

Grouping Concepts

Setting Up Grouping

Add rowGroup: true to a column definition to group by that column. The grouped column is typically hidden with hide: true.

Example: 
{{
headerName: 'Department',
field: 'department',
rowGroup: true,
hide: true
}}

Aggregation Functions

Use aggFunc to summarize numeric columns within groups. Common functions: sum, count, min, max, avg.

Example: 
{{
headerName: 'Salary',
field: 'salary',
aggFunc: 'sum'
}}

Multiple Grouping Levels

Add multiple columns with rowGroup: true to create hierarchical grouping.

Example: 
columnDefs = [
  {{ field: 'department', rowGroup: true, hide: true }},
  {{ field: 'team', rowGroup: true, hide: true }},
  {{ field: 'name' }}
];

Programmatic Grouping

Change grouping dynamically using the Column API.

TypeScript: 
groupByDept() {{
  this.gridColumnApi?.setRowGroupColumns(['department']);
}}

clearGrouping() {{
  this.gridColumnApi?.setRowGroupColumns([]);
}}

Expand/Collapse Groups

Users can click the expand arrow. You can also expand/collapse programmatically.

TypeScript: 
expandAll() {{
  this.gridApi?.expandAll();
}}

collapseAll() {{
  this.gridApi?.collapseAll();
}}

Best Practices

✓ Use grouping for summaries: Group by categorical columns, aggregate numerics.
✓ Limit grouping depth: Too many levels make data hard to navigate.
✓ Provide expand/collapse options: Let users control group visibility.
✓ Show aggregations clearly: Label aggregated values distinctly.
✓ Hide grouping columns: Set hide: true for columns used only for grouping.
Grouping Properties
rowGroup

Enable grouping by this column

hide

Hide the grouping column

aggFunc

Aggregation function (sum, avg, min, max, count)

API Methods
  • setRowGroupColumns()
  • expandAll()
  • collapseAll()
  • getRowGroupColumn()
Tip: Grouping automatically hides detail rows. Great for data exploration.
