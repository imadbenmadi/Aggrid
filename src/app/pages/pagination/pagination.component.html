Pagination Concepts

Enabling Pagination

Set pagination: true in grid properties to enable pagination. Set paginationPageSize to control rows per page.

HTML Template: 
<ag-grid-angular
  [pagination]="true"
  [paginationPageSize]="pageSize"
  ...
>
</ag-grid-angular>

Dynamic Page Size

Allow users to change the page size on the fly. The grid will automatically recalculate pagination.

TypeScript: 
updatePageSize() {{
  this.gridApi?.paginationSetPageSize(
    parseInt(this.pageSize.toString())
  );
}}

Navigation Methods

Use pagination API methods to navigate programmatically.

TypeScript: 
nextPage() {{
  this.gridApi?.paginationGoToNextPage();
}}

previousPage() {{
  this.gridApi?.paginationGoToPreviousPage();
}}

goToPage(pageNumber: number) {{
  this.gridApi?.paginationGoToPage(pageNumber);
}}

Pagination vs Virtual Scrolling

Pagination: Display one page at a time. Good for large datasets and mobile devices.
Virtual Scrolling: Continuous scrolling through all data. Good for smaller datasets with instant access.

Client-side vs Server-side Pagination

Client-side: All data loaded at once, pagination handled in the browser. Fast but uses more memory.
Server-side: Load pages on demand from the server. Better for massive datasets with limited bandwidth.

Server-side Pagination Example: 
// Load only current page from server
loadPage(page: number) {{
  this.apiService.getUsers(page, this.pageSize)
    .subscribe(data => {{
      this.rowData = data;
      this.gridApi?.paginationGoToPage(page);
    }});
}}

Best Practices

✓ Use pagination for large datasets: Keep page sizes reasonable (10-50 rows).
✓ Implement server-side pagination: For very large datasets, load pages from the server.
✓ Show row count: Display total rows to help users understand data scope.
✓ Provide page size options: Let users choose how many rows per page.
✓ Remember user preferences: Store page size in localStorage for better UX.
